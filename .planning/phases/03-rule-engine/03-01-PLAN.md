  <behavior>
    isHotLead behavior:
    - Lead with status "qualified" → true
    - Lead with status "contacted" → true
    - Lead with status "demo" → true
    - Lead with status "proposal" → true
    - Lead with status "negotiation" → true
    - Lead with status "new" → false (unqualified)
    - Lead with status "won" → false (resolved)
    - Lead with status "lost" → false (resolved)
    - Lead with status "nurture" → false (resolved)

    CrossModuleSnapshot null-module behavior:
    - snapshot.finance can be null when Finance module fails
    - snapshot.sales can be null when Sales module fails
    - snapshot.hire can be null when Hire module fails
    - snapshot.errors is always string[] (never null/undefined)
  </behavior>
  <implementation>
    Create lib/intelligence/types.ts with:

    1. HOT_STAGES constant: readonly array ["qualified", "contacted", "demo", "proposal", "negotiation"]

    2. RuleSeverity type: "critical" | "warning" | "info"

    3. RuleInsight interface:
       - rule_id: string (e.g., "R1") — required, not optional
       - severity: RuleSeverity
       - evidence: string (human-readable, e.g., "3 hot leads stalled, runway at 42 days")
       - module_tags: string[] (e.g., ["sales", "finance"])
       - generated_at: string (ISO date string)

    4. SalesSnapshot interface:
       - leads: Lead[]
       - recent_activity: LeadActivity[]

    5. HireSnapshot interface:
       - candidates: CandidateWithEvaluation[]
       - roles: Role[]

    6. FinanceSnapshot interface:
       - invoices: Invoice[]
       - expenses: Expense[]
       - runway: RunwayData | null

    7. CrossModuleSnapshot interface:
       - sales: SalesSnapshot | null
       - hire: HireSnapshot | null
       - finance: FinanceSnapshot | null
       - errors: string[]
       - generated_at: string

    8. isHotLead(lead: Lead): boolean
       - Returns true if lead.status is in HOT_STAGES
       - Import Lead from lib/sales-types.ts (NOT SalesLead or mock types)

    Run RED (fail) then GREEN (pass) cycle — commit after each phase.
