Key field references:
- Candidate stall detection: use `applied_at` (ISO string) — candidates with no `updated_at`,
  so 7-day stall = Date.now() - new Date(candidate.applied_at).getTime() > 7 * 24 * 60 * 60 * 1000
- Candidate stall only applies to status "pending" | "analyzed" (not "reviewed")
- R6 large invoice threshold: gross_amount > 10000 (Claude discretion — document in evidence string)
- R6 "upcoming payroll cost": presence of any pending invoice (status "beklemede") is the proxy
  (no explicit payroll field exists; pending invoices represent upcoming financial obligations)
- R7 no-reply: lead.last_contact_at null OR (Date.now() - new Date(last_contact_at).getTime() > 48 * 60 * 60 * 1000)
- R7 requires 2+ hot leads total, not just 1 without reply

  <behavior>
    R5 — checkR5CandidateStall(snapshot: CrossModuleSnapshot): RuleInsight | null
    - snapshot.hire null → null
    - No active roles (role.active=true) → null (no capacity gap)
    - All candidates have status "reviewed" → null
    - No candidate with applied_at > 7 days ago in "pending"/"analyzed" status → null
    - 1+ candidate stalled 7+ days AND active role exists → RuleInsight {
        rule_id: "R5", severity: "warning",
        evidence: "{N} candidate(s) pending for 7+ days with {M} open role(s)",
        module_tags: ["hire"]
      }

    R6 — checkR6InvoicePayroll(snapshot: CrossModuleSnapshot): RuleInsight | null
    - snapshot.finance null → null
    - No overdue invoices (status "gecmis") → null
    - No overdue invoice with gross_amount > 10000 → null (small overdue invoices excluded)
    - No pending invoices (status "beklemede") → null (no upcoming cost signal)
    - Large overdue invoice AND pending invoice both exist → RuleInsight {
        rule_id: "R6", severity: "critical",
        evidence: "Large overdue invoice ({amount} TL) with upcoming payment obligations",
        module_tags: ["finance"]
      }

    R7 — checkR7HotLeadsNoReply(snapshot: CrossModuleSnapshot): RuleInsight | null
    - snapshot.sales null → null
    - Fewer than 2 hot leads → null
    - 2+ hot leads BUT all have last_contact_at within 48 hours → null
    - 2+ hot leads AND at least 1 has last_contact_at null or > 48 hours ago → RuleInsight {
        rule_id: "R7", severity: "warning",
        evidence: "{N} hot lead(s) with no reply in 48+ hours out of {total} active leads",
        module_tags: ["sales"]
      }

    - Accepts CrossModuleSnapshot
    - Calls all 7 rules
    - Filters out null results
    - Returns RuleInsight[] (may be empty array)
    - Never throws — individual rule errors are caught, logged to console.error, excluded from results
  </behavior>
