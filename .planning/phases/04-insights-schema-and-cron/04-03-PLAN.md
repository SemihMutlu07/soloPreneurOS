    ```typescript

      insights_generated: 0,
      insights_upserted: 0,
      errors: [] as string[],

    ```

    **Step 3 — Pipeline (inside try/catch):**
    Execute in sequence:
    1. `const snapshot = await buildCrossModuleSnapshot()` — call Phase 2 aggregator. If snapshot.errors has entries, append them to result.errors (do not abort).
    2. `const candidates = await runRuleEngine(snapshot)` — call Phase 3 rule engine. Assign result.insights_generated = candidates.length.
    3. `const persistResult = await persistInsights(candidates)` — call Plan 02 persist function. Assign result.insights_upserted = persistResult.upserted. Append any persistResult.errors to result.errors.

    **Step 4 — Return:**
    ```typescript
    return NextResponse.json(result);
    ```

    **Error handling:**
    - Wrap the entire pipeline in a top-level try/catch. On unhandled error, return `NextResponse.json({ error: message, ...result }, { status: 500 })`.
    - Individual item errors are already handled inside persistInsights — no per-candidate try/catch needed in the cron route itself.

    **Imports needed:**
    - `import { NextResponse } from "next/server"`
    - `import { buildCrossModuleSnapshot } from "@/lib/..."` (find exact path from Phase 2 SUMMARY)
